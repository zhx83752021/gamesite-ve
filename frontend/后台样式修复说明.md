# 后台管理系统样式修复说明

## 修复的问题

### 1. 下拉菜单文字垂直不居中

**问题原因**：Element Plus 下拉菜单项默认样式的 line-height 导致文字无法垂直居中

**解决方案**：

- 为 `.admin-layout .el-dropdown-menu__item` 添加 `display: flex` 和 `align-items: center`
- 设置 `line-height: normal` 覆盖默认行高
- 优化内边距为 `8px 16px`

### 2. 按钮文字颜色问题（白色看不到）

**问题原因**：全局样式设置了深色主题（`body { color: #E0E7FF }`），导致后台白色背景上的文字继承了浅色，无法看清

**解决方案**：

- 为后台区域重置基础文字颜色为 `#303133`（深色）
- 明确设置各种按钮类型的文字颜色：
  - 常规按钮：深色文字 `#303133`
  - Primary/Success/Warning/Danger/Info 按钮：白色文字 `#ffffff`
  - Link 类型按钮：使用对应的主题色（蓝色/黄色/红色）

### 3. 其他组件样式优化

#### 表格

- 设置表头和单元格文字颜色为 `#303133`

#### 输入框和选择器

- 输入文字颜色：`#606266`
- 占位符颜色：`#C0C4CC`
- 下拉选项颜色：`#606266`
- 选中选项颜色：`#409EFF`

#### 卡片和对话框

- 标题和内容文字使用标准深色
- 确保在白色背景上清晰可读

#### 分页器

- 按钮和页码文字颜色：`#606266`

#### 面包屑

- 普通文字：`#606266`
- 悬停链接：`#409EFF`

#### 表单

- 标签文字：`#606266`

### 4. 用户下拉菜单优化

**优化内容**：

- 添加悬停效果（背景色变化）
- 优化内边距和圆角
- 确保用户名、头像、箭头图标完美垂直对齐

## 技术要点

### CSS 优先级策略

使用 `.admin-layout` 作为命名空间前缀，配合 `!important` 确保样式优先级高于全局深色主题样式

### 颜色规范

遵循 Element Plus 的颜色体系：

- 主文字：`#303133`
- 次要文字：`#606266`
- 占位符/禁用：`#909399`, `#C0C4CC`
- 边框：`#DCDFE6`, `#E4E7ED`
- 主题色：`#409EFF`（蓝）, `#67C23A`（绿）, `#E6A23C`（黄）, `#F56C6C`（红）

## 注意事项

- 修改仅影响后台管理系统（`.admin-layout` 内），不影响前台页面的深色主题
- 所有样式修复都是通过 CSS 覆盖实现，未修改组件逻辑
- 保持了 Element Plus 组件的原有交互行为
