## 五、风险管理

### 5.1 技术风险

| 风险             | 概率 | 影响 | 应对措施                     |
| ---------------- | ---- | ---- | ---------------------------- |
| API 性能问题     | 中   | 高   | 提前做压力测试，优化查询     |
| 数据库设计缺陷   | 低   | 高   | 充分的需求分析，预留扩展字段 |
| 第三方服务不稳定 | 低   | 中   | 使用多个备选方案             |
| 浏览器兼容性问题 | 中   | 中   | 提前做兼容性测试             |

### 5.2 进度风险

| 风险         | 概率 | 影响 | 应对措施               |
| ------------ | ---- | ---- | ---------------------- |
| 需求变更     | 中   | 高   | 锁定需求，控制变更     |
| 开发延期     | 中   | 高   | 预留缓冲时间，及时调整 |
| 测试时间不足 | 低   | 中   | 边开发边测试           |

### 5.3 资源风险

| 风险         | 概率 | 影响 | 应对措施               |
| ------------ | ---- | ---- | ---------------------- |
| 免费额度不足 | 低   | 中   | 监控使用量，及时升级   |
